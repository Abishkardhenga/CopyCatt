### Recommended Hero Background System (Use React Bits Background Studio)

For the hero background, use a **React Bits Background Studio-style animated background** instead of a static gradient. The goal is to recreate the premium, futuristic feel of Arc Genesis / Winterfell while keeping it unique to CopyCatt.

**Preferred background behavior:**
- Dark cinematic base with soft glow layers
- Interactive particle / beam / grid motion
- Subtle mouse-follow or hover response
- Motion should feel premium, not noisy
- Background must stay behind headline and CTA without hurting readability

**Best fit for CopyCatt:**
- Use a **dark particle field or beam-based animated background**
- Add **lime-tinted glow accents** so the effect matches the CopyCatt brand
- Keep overall intensity low so the cat mascot, headline, and waitlist form stay dominant

**Background direction:**
- Base: near-black / deep charcoal
- Motion: slow floating particles, soft lines, glow haze, or grid drift
- Interaction: cursor movement should slightly push particles or shift the glow
- Layering: background on lowest layer, glow overlay in middle, hero card/content on top

**Why this works:**
It gives the landing page the same premium Web3 energy as Arc Genesis and Winterfell, but keeps the result original and easier to implement. It also makes the hero feel alive when users move their cursor, which increases perceived polish.

---

### React Bits Background Implementation Notes

**Recommended component setup:**
- Use a React Bits animated background component as the base layer of the hero section
- Wrap the hero content in a relative container
- Place the animated background absolutely with full inset coverage
- Add a soft radial gradient overlay above the background
- Keep the content container above both with `z-index`

**Suggested visual settings:**
- Background opacity: `0.35 - 0.55`
- Blur/glow intensity: low to medium
- Animation speed: slow
- Interaction strength: subtle
- Particle density: medium-low
- Accent glow color: lime/green with optional cyan support

**Performance rules:**
- Reduce motion and density on mobile
- Provide a static gradient fallback for low-end devices
- Avoid high-frequency particle counts
- Keep animation GPU-friendly

**Implementation structure:**
```md
Hero Section
├── Animated Background (React Bits)
├── Gradient / noise overlay
├── Mascot or dashboard visual
└── Hero copy + CTA
```

### Developer Handoff Notes

**Tell the developer to build the hero like this:**
